=== OVERVIEW ===
`useClipboard` provides access to the system clipboard, allowing your app to copy text to and paste text from the clipboard. It returns `copy` and `paste` functions, plus a `copied` boolean that temporarily turns `true` after a successful copy operation, making it easy to show "Copied!" feedback to the user.

=== API / SYNTAX ===
```tsx
import { useClipboard } from '@ilovereact/core';

const { copy, paste, copied } = useClipboard();
```

**Parameters:**
None — `useClipboard()` takes no arguments.

**Return Value:**

Returns an object with the following properties:

| Property | Type | Description |
|----------|------|-------------|
| `copy` | `(text: string) => Promise<void>` | Copies the given text to the system clipboard. Returns a promise that resolves when the operation completes. |
| `paste` | `() => Promise<string>` | Reads the current clipboard contents and returns it as a string. |
| `copied` | `boolean` | `true` for 2 seconds after a successful `copy()` call, then resets to `false`. Use this to show temporary "Copied!" UI feedback. |

=== EXAMPLES ===

Example 1: Basic Copy Button with Feedback
---
```tsx
import { Box, Text, Pressable } from '@ilovereact/core';
import { useClipboard } from '@ilovereact/core';

function ShareLink() {
  const { copy, copied } = useClipboard();
  const shareUrl = 'https://example.com/share/abc123';

  return (
    <Box padding={20}>
      <Text fontSize={16}>Share this link:</Text>
      <Text fontSize={14} color="#888888">{shareUrl}</Text>
      <Pressable onPress={() => copy(shareUrl)}>
        <Box
          marginTop={10}
          padding={10}
          backgroundColor={copied ? '#00ff00' : '#0088ff'}
        >
          <Text fontSize={14}>{copied ? 'Copied!' : 'Copy Link'}</Text>
        </Box>
      </Pressable>
    </Box>
  );
}
```

Example 3: Paste from Clipboard
---
```tsx
import { Box, Text, Pressable } from '@ilovereact/core';
import { useClipboard } from '@ilovereact/core';

function ImportFromClipboard() {
  const { paste } = useClipboard();
  const [content, setContent] = React.useState('');

  const handlePaste = async () => {
    const text = await paste();
    setContent(text);
  };

  return (
    <Box padding={20}>
      <Pressable onPress={handlePaste}>
        <Box padding={10} backgroundColor="#0088ff">
          <Text fontSize={14}>Paste from Clipboard</Text>
        </Box>
      </Pressable>
      {content && (
        <Box marginTop={20}>
          <Text fontSize={16}>Pasted content:</Text>
          <Text fontSize={14}>{content}</Text>
        </Box>
      )}
    </Box>
  );
}
```

Example 4: Copy with Hotkey
---
```tsx
import { Box, Text } from '@ilovereact/core';
import { useClipboard, useHotkey } from '@ilovereact/core';

function TextDisplay({ text }: { text: string }) {
  const { copy, copied } = useClipboard();

  // Ctrl+C copies the displayed text
  useHotkey('ctrl+c', () => copy(text));

  return (
    <Box padding={20}>
      <Text fontSize={16}>{text}</Text>
      {copied && (
        <Box marginTop={10}>
          <Text fontSize={12} color="#00ff00">Copied to clipboard!</Text>
        </Box>
      )}
      <Text fontSize={12} color="#888888" marginTop={10}>
        Press Ctrl+C to copy
      </Text>
    </Box>
  );
}
```

Example 5: Advanced Copy with Error Handling
---
```tsx
import { Box, Text, Pressable } from '@ilovereact/core';
import { useClipboard } from '@ilovereact/core';

function SecureDataCopy({ data }: { data: string }) {
  const { copy, copied } = useClipboard();
  const [error, setError] = React.useState<string | null>(null);

  const handleCopy = async () => {
    try {
      await copy(data);
      setError(null);
    } catch (err) {
      setError('Failed to copy to clipboard');
      console.error('Clipboard error:', err);
    }
  };

  return (
    <Box padding={20}>
      <Pressable onPress={handleCopy}>
        <Box padding={10} backgroundColor="#0088ff">
          <Text fontSize={14}>{copied ? 'Copied!' : 'Copy Data'}</Text>
        </Box>
      </Pressable>
      {error && (
        <Box marginTop={10}>
          <Text fontSize={12} color="#ff0000">{error}</Text>
        </Box>
      )}
    </Box>
  );
}
```

=== PLATFORM NOTES ===

**Love2D:**
- Clipboard operations are implemented via RPC calls to Lua-side functions: `clipboard:write` and `clipboard:read`
- The Lua runtime uses Love2D's `love.system.getClipboardText()` and `love.system.setClipboardText()` under the hood
- Clipboard access is synchronous on the Lua side but wrapped in promises on the React side for consistency with web APIs

**Web:**
- Uses the browser's `navigator.clipboard` API (Clipboard API)
- Requires HTTPS or localhost for security reasons (browsers block clipboard access on insecure origins)
- Some older browsers may not support the Clipboard API — consider feature detection if targeting legacy environments
- Browser extensions or security policies may block clipboard access — the promise will reject if access is denied

**Other Targets:**
- Terminal, Neovim, ComputerCraft, Hammerspoon, and AwesomeWM do not currently support `useClipboard`
- If clipboard functionality is needed for these targets, it must be implemented at the platform level via custom RPC endpoints

=== CRITICAL RULES ===

- **`copy()` and `paste()` are async** — always `await` these calls or use `.then()` to handle the promise. Don't treat them as synchronous functions.
- **`copied` resets after 2 seconds** — the feedback boolean is automatically cleared via a timer. You don't need to manually reset it.
- **Timer cleanup is automatic** — if the component unmounts before the 2-second timer expires, the timer is cleared. No memory leaks.
- **Handle promise rejections** — clipboard operations can fail (e.g., permissions denied, clipboard unavailable). Wrap calls in `try/catch` or provide `.catch()` handlers for production apps.
- **Empty clipboard returns empty string** — if the clipboard is empty or contains non-text data, `paste()` returns `''` (not `null` or `undefined`).
- **Only works with text** — `useClipboard` does not support images, files, or rich content. For those, you'd need to use platform-specific APIs directly.
- **Multiple `copy()` calls reset the timer** — if you call `copy()` again before the 2-second timer expires, the timer resets and `copied` stays `true` for another 2 seconds.
