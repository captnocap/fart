=== OVERVIEW ===
Registers component data in a global registry (`globalThis.__debug`) that the inspector console can read and evaluate. Any data registered is accessible via JavaScript evaluation in the console or as watch expressions. Data is automatically updated on every render and cleaned up on unmount. Essential for debugging component state, props, and derived values in real-time without modifying render output.

=== API / SYNTAX ===
Import:
```typescript
import { useDebug } from '@ilovereact/core';
```

Signature:
```typescript
useDebug(key: string, data: any): void
```

Parameters:
- `key` (string): Unique identifier for the debug entry, usually the component name. Must be unique across all active components.
- `data` (any): Any serializable data to expose. Objects, arrays, primitives, functions all work. Updated on every render.

Return value: void

Companion utilities:
```typescript
// Non-hook: read all debug data (for tooling)
getDebugData(): Record<string, any>

// Non-hook: imperatively register debug data outside React
// Returns unregister function
registerDebug(key: string, data: any): () => void
```

=== EXAMPLES ===
Example 1: Basic component state debugging
---
function PlayerHUD({ player }) {
  const [health, setHealth] = useState(100);
  const [mana, setMana] = useState(50);

  useDebug('PlayerHUD', { health, mana, player });

  return (
    <Box>
      <Text fontSize={16}>HP: {health}</Text>
      <Text fontSize={16}>MP: {mana}</Text>
    </Box>
  );
}

// In inspector console (F12):
// > __debug
// { PlayerHUD: { health: 100, mana: 50, player: {...} } }
// > __debug['PlayerHUD'].health
// 100
---

Example 2: Watch expressions for live monitoring
---
function GameLoop({ deltaTime }) {
  const fps = Math.round(1 / deltaTime);
  const frameCount = useRef(0);

  useEffect(() => {
    frameCount.current++;
  });

  useDebug('GameLoop', { fps, frameCount: frameCount.current, deltaTime });

  return null;
}

// In inspector console:
// :watch __debug['GameLoop'].fps
// :watch __debug['GameLoop'].frameCount
---

Example 3: Computed values and complex state
---
function DataDashboard({ users, orders }) {
  const stats = useMemo(() => ({
    totalUsers: users.length,
    activeUsers: users.filter(u => u.active).length,
    totalRevenue: orders.reduce((sum, o) => sum + o.total, 0),
    avgOrderValue: orders.length ?
      orders.reduce((sum, o) => sum + o.total, 0) / orders.length : 0,
  }), [users, orders]);

  useDebug('Dashboard', { stats, userCount: users.length });

  return <Box>...</Box>;
}

// In inspector console:
// > __debug['Dashboard'].stats.totalRevenue
// 15420.50
---

Example 4: Multiple debug entries from different components
---
function App() {
  return (
    <Box>
      <PlayerHUD player={player} />
      <Inventory items={items} />
      <QuestLog quests={quests} />
    </Box>
  );
}

function PlayerHUD({ player }) {
  useDebug('PlayerHUD', { health: player.health, level: player.level });
  return <Box>...</Box>;
}

function Inventory({ items }) {
  useDebug('Inventory', { itemCount: items.length, totalWeight: items.reduce((s, i) => s + i.weight, 0) });
  return <Box>...</Box>;
}

function QuestLog({ quests }) {
  useDebug('QuestLog', { active: quests.filter(q => q.active), completed: quests.filter(q => q.completed) });
  return <Box>...</Box>;
}

// In inspector console:
// > __debug
// {
//   PlayerHUD: { health: 85, level: 12 },
//   Inventory: { itemCount: 23, totalWeight: 45.2 },
//   QuestLog: { active: [...], completed: [...] }
// }
---

Example 5: Imperative registration outside React
---
// Game service or event loop
const gameState = {
  running: true,
  paused: false,
  tickCount: 0,
};

const unregister = registerDebug('GameState', gameState);

function tick() {
  gameState.tickCount++;
  // Update reference — useDebug auto-updates, but imperative registration is static
  globalThis.__debug['GameState'] = gameState;
  requestAnimationFrame(tick);
}

// Later: cleanup
unregister();
---
Platforms: All

=== CRITICAL RULES ===
- Keys must be unique. Duplicate keys overwrite each other (last registration wins).
- Data is updated on every render. Don't register expensive computations without memoization.
- Data is NOT reactive in the console. Type the expression again to see updated values, or use `:watch` for auto-refresh.
- Automatically unregisters on component unmount. No manual cleanup needed for hook usage.
- Use `registerDebug` for non-React code (game loops, services, etc.). Remember to call the unregister function when done.
- Inspector console evaluates in the QuickJS/V8 context. Standard JavaScript eval rules apply.
- Do NOT register circular references or non-serializable objects (DOM nodes, class instances with methods) — they may crash the console or produce unhelpful output.
- Registered data lives on `globalThis`, so it persists across hot reloads in dev mode. Clear manually if needed: `globalThis.__debug = {}`.
