=== OVERVIEW ===
Converts Unicode art strings or built-in symbol names into colored `<Box>` grids that render consistently across all targets. Love2D's default font lacks Unicode symbols (▶ ⏸ ● ✓ arrows, dingbats, geometric shapes), so this hook generates Box-based geometry as a replacement. Accepts either a built-in symbol name or custom string art lines. Any non-space character becomes a filled pixel. Memoized for performance.

=== API / SYNTAX ===
Import:
```typescript
import { usePixelArt, PixelArt, getPixelArtSymbols } from '@ilovereact/core';
```

Hook signature:
```typescript
usePixelArt(
  art: string | string[],
  options?: PixelArtOptions
): React.ReactElement
```

Parameters:
- `art` (string | string[]): Built-in symbol name (e.g., 'play', 'pause', 'heart') OR array of string art lines. Any non-space character = filled pixel.
- `options` (PixelArtOptions, optional):
  - `size` (number, default: 4): Pixel size in logical units
  - `color` (string, default: '#ffffff'): Single fill color for all pixels
  - `colors` (string[], optional): Per-row fill colors. Overrides `color`. Falls back to `color` for rows beyond array length.
  - `gap` (number, default: 0): Gap between pixels in logical units

Return value: React element (nested `<Box>` grid)

Component signature:
```typescript
<PixelArt
  art={string | string[]}
  size?: number
  color?: string
  colors?: string[]
  gap?: number
  style?: Style
/>
```

Utility:
```typescript
getPixelArtSymbols(): string[]
// Returns array of available built-in symbol names
```

Built-in symbols:
play, pause, stop, skip-next, skip-prev, check, close, arrow-up, arrow-down, arrow-left, arrow-right, circle, circle-outline, dot, plus, minus, heart, star, volume, mute, chevron-right, chevron-left, menu

Example 1: Built-in symbol with default options
---
function PlayButton() {
  const playIcon = usePixelArt('play', { size: 4, color: '#ffffff' });

  return (
    <Pressable onPress={() => console.log('play')}>
      <Box style={{ padding: 8 }}>
        {playIcon}
      </Box>
    </Pressable>
  );
}

Example 5: Animated icon with state changes
---
function VolumeControl({ volume, onToggleMute }) {
  const [muted, setMuted] = useState(false);

  const icon = usePixelArt(muted ? 'mute' : 'volume', {
    size: 4,
    color: muted ? '#888888' : '#ffffff',
  });

  const handlePress = () => {
    setMuted(!muted);
    onToggleMute(!muted);
  };

  return (
    <Pressable onPress={handlePress}>
      <Box style={{ padding: 8 }}>{icon}</Box>
    </Pressable>
  );
}

=== CRITICAL RULES ===
- NEVER put Unicode symbols directly in `<Text>` — use this hook instead. The linter enforces this via `no-unicode-symbol-in-text`.
- Any non-space character in the art string = filled pixel. Spaces = transparent.
- Built-in symbol names are case-sensitive. Use `getPixelArtSymbols()` to list available names.
- Result is memoized. Art and options changes trigger re-render, but repeated calls with same args are cheap.
- Colors array applies per-row. If fewer colors than rows, remaining rows use `color` fallback.
- Pixel `size` is in logical units, not physical pixels. Same value renders consistently across Love2D, terminal, and web.
- Gap is applied between pixels in both dimensions. Use `gap: 1` or `gap: 2` for clearer visual separation.
- Custom art strings can use any character. Unicode works in the art definition (since it becomes Box geometry), just not in final Text output.
- Component form (`<PixelArt>`) accepts `style` prop to wrap in a styled container. Hook form returns bare element.
- Do NOT manually construct Box grids for icons — use this hook. It handles memoization, color interpolation, and edge cases.
