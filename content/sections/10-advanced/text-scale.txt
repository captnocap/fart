=== OVERVIEW ===
Global Text Scale allows runtime text zooming via Ctrl+= and Ctrl+- (like browser zoom). The global multiplier ranges from 0.5 to 3.0 and is managed by lua/measure.lua. Individual subtrees can override the global scale by setting the textScale style prop, which walks up the ancestor chain to find the nearest override. All text measurement is re-cached when scale changes, triggering automatic layout reflow.

=== API / SYNTAX ===
Keyboard Shortcuts:
  Ctrl+=       Increase global text scale (zoom in)
  Ctrl+-       Decrease global text scale (zoom out)
  Ctrl+0       Reset global text scale to 1.0

Style Property:
  textScale: number      Per-subtree text scale override (inherits to descendants)

Lua API (measure.lua):
  Measure.setTextScale(scale)         Set global scale, clamps to [0.5, 3.0], clears cache
  Measure.getTextScale()              Get current global scale
  Measure.resolveTextScale(node)      Walk ancestor chain, return effective scale for node
  Measure.scaleFontSize(fontSize, node)   Apply resolved scale: floor(fontSize * resolveTextScale(node))

How It Works:
  1. Global scale is a single float in measure.lua (default 1.0)
  2. When Ctrl+=/- is pressed, main.lua calls Measure.setTextScale(newScale)
  3. setTextScale clamps to [0.5, 3.0] and calls Measure.clearCache()
  4. Next layout pass re-measures all text at new scale
  5. If a node has textScale style prop, that value overrides global scale for that subtree
  6. Measure.resolveTextScale(node) walks up tree looking for textScale prop, falls back to global
  7. Painter applies resolved scale when rendering text

=== EXAMPLES ===
Example 1: Global Zoom Controls
---
// main.lua or init.lua:
function love.keypressed(key)
  if love.keyboard.isDown("lctrl") or love.keyboard.isDown("rctrl") then
    local measure = require("lua.measure")
    local currentScale = measure.getTextScale()

    if key == "=" or key == "+" then
      measure.setTextScale(currentScale + 0.1)  -- zoom in
      tree.markDirty()  -- trigger re-layout
      return
    elseif key == "-" then
      measure.setTextScale(currentScale - 0.1)  -- zoom out
      tree.markDirty()
      return
    elseif key == "0" then
      measure.setTextScale(1.0)  -- reset
      tree.markDirty()
      return
    end
  end
end

// User presses Ctrl+= three times:
// Scale: 1.0 -> 1.1 -> 1.2 -> 1.3
// All text in app grows proportionally
---
Platforms: Love2D

<Box width="100%" height="100%">

    Normal text at global scale
  </Text>

  <Box textScale={0.8}>

    <Text fontSize={20}>
      Also scaled to 80% (effective fontSize: 16px)
    </Text>
  </Box>

    <Text fontSize={16}>
      Larger text (150% of global scale)
    </Text>
  </Box>

Example 3: Accessibility Zoom for Entire App
---
// App wrapper with optional user-controlled zoom:
function App() {
  const [userZoom, setUserZoom] = useState(1.0);

  return (
    <Box width="100%" height="100%" textScale={userZoom}>
      <Navbar>
        <Text fontSize={14}>File</Text>
        <Text fontSize={14}>Edit</Text>
        <Button onPress={() => setUserZoom(userZoom + 0.1)}>
          <Text fontSize={12}>A+</Text>
        </Button>
        <Button onPress={() => setUserZoom(userZoom - 0.1)}>
          <Text fontSize={12}>A-</Text>
        </Button>
      </Navbar>

      <Content>
        {/* All text here inherits textScale={userZoom} */}
      </Content>
    </Box>
  );
}

// User clicks A+ button -> userZoom = 1.1
// All text in app (except buttons themselves) scales to 110%
// Global Ctrl+/- still works independently
---

Example 5: Dynamic Scale Based on Window Size
---
function ResponsiveApp() {
  const { width } = useWindowDimensions();

  // Auto-scale text on small windows
  const autoScale = width < 800 ? 0.85 : 1.0;

  return (
    <Box width="100%" height="100%" textScale={autoScale}>
      <Text fontSize={24}>Heading</Text>
      <Text fontSize={16}>Body text</Text>
    </Box>
  );
}

// Window 1200px wide:
//   autoScale = 1.0, text renders at 24px and 16px

// Window resized to 600px:
//   autoScale = 0.85, text renders at 20px and 13px

// User can still Ctrl+/- to adjust further:
//   Ctrl+= with autoScale=0.85 and global=1.2 -> effective 0.85*1.2 = 1.02
---

=== CRITICAL RULES ===
- Global text scale is clamped to [0.5, 3.0] — values outside this range are rejected
- Changing scale clears the measurement cache — all text is re-measured on next layout pass, can cause frame stutter on large trees
- textScale style prop is NOT inherited automatically — it must be explicitly set on container nodes, then descendants resolve it via ancestor walk
- textScale multiplies with global scale — textScale={0.8} with global 1.5 results in effective 1.2
- Font size is floored after scaling — 16 * 0.85 = 13.6 -> 13px (can cause aliasing at small sizes)
- Ctrl+0 resets ONLY global scale — per-subtree textScale props are unaffected
- Scale changes do NOT trigger React re-render — only layout reflow (call tree.markDirty() after setTextScale)
- DevTools console can inspect resolved scale — :lua Measure.resolveTextScale(node)
- Text scale affects layout dimensions — larger text causes wrapping, container growth, scrollbar appearance
- Scale changes do NOT affect non-text elements — Box sizes, Image dimensions, borders, etc. are unaffected
- Painter uses Measure.scaleFontSize(fontSize, node) — never multiply fontSize manually in components
- Zero or negative textScale values are undefined behavior — always use positive floats
- textScale does NOT apply to DevTools UI — inspector and console text is always 11px regardless of global scale
- Scale resolution walks up to root — deepest override wins, not nearest ancestor with explicit value
- Measurement cache keys include resolved scale — same text at different scales produces different cache entries
