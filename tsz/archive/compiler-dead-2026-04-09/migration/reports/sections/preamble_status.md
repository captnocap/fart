### Diff analysis

Two sources of divergence in the first hunk:

1. **Harness file separators**: Legacy concatenation prepends `// ── app.zig ──` per file (run_parity.js L73). Atom output does not include these separators. This is a harness artifact, not a real preamble difference.

2. **Banner format**: Legacy forge emits `Generated by tsz compiler — s01b_counter [app.zig]` while atom a001 emits `Generated by tsz compiler (Zig) — do not edit`. The monolith `emit/preamble.js` matches the atom format (`(Zig) — do not edit`). The built forge binary may be using an older Smith bundle that predates the current banner string in `emit/preamble.js`.

Neither divergence indicates a preamble atom code defect. The a002/a003 changes (smith_log, evalLuaMapData, hasLuaMaps condition, core.zig relocation) are structurally verified against the monolith JS source. Full byte-parity requires rebuilding forge with the current Smith bundle (`zig build forge`).

Re-verified with fixed harness (commit 011a3e34) and rebuilt forge (`zig build forge`). Legacy hash changed (forge rebuild took effect) but banner diff persists.

### Resolution

All conformance carts route through the lua-tree path (`emit.js` L3: `ctx._luaRootNode`), which uses `lua_tree_emit.js` (different preamble). Fixed by patching `parity_intercept.js` to temporarily suppress `ctx._luaRootNode` in parity mode, forcing legacy through `emitPreamble()` — the actual compare surface for atoms a001-a003.

After this fix, preamble parity lines 1-25 are **IDENTICAL** between legacy and atom. First diff is at line 26 (state manifest section comment dash length — a004 territory, not preamble).
